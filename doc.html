Huertos + Tecnología + Comunidad

Qué és?


Es una plataforma web diseñada para conectar huertos urbanos, que quieren darse a conocer a la sociedad, con todas aquellas personas que desean formar parte de este movimiento.

La aplicacion permite a los urbanitas buscar huertos cercanos, ver como funcionan, unirse a ellos, establecer un contacto y participar en sus actividades. Los coordinadores tendran la posibilidad de dar a conocer su huerto y crear actividades que atraigan a nuevos urbanitas para facilitar la conexion de su huerto con las personas.

Como trabajamos?

Fases del proceso

  • Idea

    En cada sprint la empresa nos traslada el objetivo que se marcan para esa semana y nos explican el concepto.

  • Mockup

    A partir de este concepto, se genera un diseño de referencia para el equipo de desarrollo.

  • Especificaciones

    Este mockup se desglosa en tareas que se reparten entre los miembros del equipo.

  • Desarrollo

    Durante el sprint, se llevan a cabo las tareas definidas.

  • Validación

    Finalmente, se presenta el desarrollo a la empresa y ésta la valida de acuerdo con los criterios de aceptación.

Lógica de negocio (Interna)

Entities, Services, Controllers & Routing

EventBundle

Este bundle encapsula la funcionalidad relacionada con la entidad Event. Esta entidad se utiliza para gestionar los eventos o actividades que pueden crear los coordinadores de un huerto. Se han declarado las funcionalidades generales en el servicio EventService para optimizar el código.Las plantillas se dividen en dos partes, la de creación y gestión y la del perfil.

* * *

HomeBundle

Este bundle contiene la parte principal de la aplicación desde la que se puede acceder a todos los demás. También econtramos la parte del buscador, la parte de explicación sobre cómo funciona la aplicación y el porque de ella.

* * *

OrchardBundle

Este bundle sin duda es el más importante de Huert On ya que contiene toda la parte de creación, modificación, gestión y visualización de un huerto. Por otro lado, también contiene un controlador y un conjunto de plantillas para gestionar las sugerencias y email de contacto. Se han declarado las funcionalidades generales en el servicio OrchardService para optimizar el código. Las rutas se dividen en tres partes, la creación, visualización, emails y subida de ficheros.

* * *

UserBundle

Al tener integrado el FOSUserBundle utilizamos este bundle para sobreescribir las vistas y la entidad user de esta manera hemos podio añadir el campo apellidos y google id para integrar el login de Google gracias al bundle HWIOAuthBundle.

Puesta en producción

Hemos usado dos proveedores de VPS para desplegar nuestro proyecto. El servidor de preproducción esta alojado en una EC2 de AWS y el de producción en OVH.

Para organizar el repositorio nos hemos basado en el modelo de ramas git-flow y hemos usado el gestor de repositorios SmartGit.

//Actualizamos el repositorio
git checkout -f
git pull origin master

//Asignamos los permisos
chown -R www-data:www-data repo
setfacl -R -m u:www-data:rX repo
cd repo
setfacl -R -m u:www-data:rwX var/cache/ var/logs/
setfacl -dR -m u:www-data:rwX var/cache/ var/logs/

//Añadimos la variable de entorno SYMFONY_ENV
export SYMFONY_ENV=prod

//Preparamos el proyecto
composer install --no-dev --optimize-autoloader
php bin/console doctrine:schema:update --force
php bin/console cache:clear --env=prod

Hemos automatizado este proceso creando el script deploy.sh
